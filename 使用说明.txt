DEM 温度和降水降尺度脚本使用说明
================================

一、脚本是否可以独立运行

dem_downscale.py 是独立 Python 脚本，不依赖本项目中的其他 Python 文件。
将以下两个文件放到任意目录即可配置环境并运行：

1. dem_downscale.py
2. requirements.txt

脚本依赖 Python 标准库以及 numpy、rasterio、affine、pyproj、tqdm。


二、推荐运行环境

推荐使用：

Python 3.9，64 位

脚本已在以下环境验证：

Python 3.9.19
numpy 1.26.4
rasterio 1.3.4
affine 2.4.0
pyproj 3.4.1
tqdm 4.65.2

由于 rasterio、pyproj 涉及 GDAL 和 PROJ，推荐使用 Conda 创建独立环境。


三、使用 Conda 配置环境（推荐）

打开 Anaconda Prompt 或 Miniconda Prompt，依次运行：

conda create -n dem_downscale python=3.9 -y
conda activate dem_downscale
python -m pip install --upgrade pip
python -m pip install -r requirements.txt

确认依赖安装成功：

python -c "import numpy, rasterio, affine, pyproj, tqdm; print('环境正常')"


四、仅使用 pip 配置环境

确认计算机已经安装 64 位 Python 3.9，然后在脚本目录运行：

python -m venv .venv

Windows PowerShell 激活环境：

.\.venv\Scripts\Activate.ps1

Windows CMD 激活环境：

.\.venv\Scripts\activate.bat

安装依赖：

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

如果 rasterio 或 pyproj 安装失败，建议改用上面的 Conda 方法。


五、准备输入目录

温度和降水分别放在两个目录中，例如：

D:\climate\temperature
D:\climate\precipitation

文件名需要包含年月，支持的常用格式包括：

温度：
tem202501四川.tif
tem_202501四川.tif
tem-202501四川.tif

降水：
pre202501四川.tif
pre_202501四川.tif
pre-202501四川.tif
precip202501四川.tif

其中年月必须是 YYYYMM 格式，月份范围为 01 到 12。

同一个月份可以只有温度或只有降水。脚本会报告缺测变量，并继续处理存在的变量。


六、配置脚本路径

使用文本编辑器打开 dem_downscale.py，在文件底部的：

if __name__ == "__main__":

代码块中修改以下四个路径：

temp_input_dir = Path(r"D:\climate\temperature")
precip_input_dir = Path(r"D:\climate\precipitation")
dem_file = Path(r"D:\climate\dem_30m.tif")
output_dir = Path(r"D:\climate\output")

路径前面的 r 不要删除。

DEM 是输出参考栅格。输出文件的 CRS、仿射变换、分辨率、行列数和空间范围均以 DEM 为准。


七、输入数据要求

1. DEM、温度和降水必须是可由 rasterio 读取的单波段 GeoTIFF。
2. 所有输入必须带有有效 CRS 和可逆的仿射变换。
3. 温度或降水投影到 DEM 坐标系后必须与 DEM 有空间重叠。
4. DEM 单位应为米。
5. 温度单位应为摄氏度。
6. 降水单位应为 mm/month。
7. 粗分辨率数据通常为 1 km，但脚本实际依据输入栅格自身的 CRS 和仿射变换进行映射。
8. 温度和降水输入可以使用与 DEM 不同的 CRS，脚本会在计算时转换坐标。


八、运行脚本

进入 dem_downscale.py 所在目录，然后运行：

python dem_downscale.py

Windows 下如果中文输出乱码，可以运行：

python -X utf8 dem_downscale.py

也可以将日志保存为 UTF-8 文件：

python -X utf8 dem_downscale.py > batch_run.log 2>&1


九、输出目录

脚本会在 output_dir 下自动建立：

output_dir\temperature
output_dir\precipitation

输出文件命名格式为：

原输入文件名_30m.tif

例如：

temperature\tem202501四川_30m.tif
precipitation\pre_202501四川_30m.tif

同名输出文件会被覆盖。


十、运行结果和质量检查

每个月处理完成后，脚本会打印：

1. 温度递减率或降水高程梯度。
2. 回归样本数及是否发生回退。
3. 温度修正范围或降水权重范围。
4. 父像元均值和降水总量守恒误差。
5. 降水负值像元数量。
6. 输出 CRS、仿射变换、行列数和分辨率是否与 DEM 一致。

温度输出在每个原始粗像元内保持均值一致。
降水输出在每个原始粗像元内保持平均降水深度一致，并检查对应的面积总量守恒。


十一、资源需求

脚本采用分块读取，不会一次将整个 30 m DEM 加载到内存。
但 30 m 全区域 GeoTIFF 输出通常较大，请预留足够磁盘空间。
批处理耗时主要取决于 DEM 像元数量、月份数量和磁盘读写速度。
